## Why
当前若运行环境缺少 `huggingface_hub` 等关键依赖,程序启动或首次下载时才会报错,用户体验差。
希望在程序启动(后端)即检查关键依赖是否齐备,缺失时通过 WebUI 明确提示用户安装,
避免"点了下载才发现没法下"。

## What Changes
### 1. 后端启动检查关键依赖
在 `app.py` 启动时探测关键第三方依赖:`fastapi`、`uvicorn`、`huggingface_hub`、`tqdm`
(即下载功能实际依赖的库)。探测结果通过 `GET /api/health`(或并入现有状态接口)返回,
列出缺失的依赖名与建议安装命令 `pip install -r requirements.txt`。
后端即使缺失依赖也 SHALL 正常启动(页面仍可打开),仅在前端提示,不硬性崩溃。

### 2. 前端进入页面提示
前端进入页面时静默请求健康检查接口;若存在缺失依赖,顶部显示一条**红色非阻断提示条**,
列出缺失包名与 `pip install -r requirements.txt` 命令,提示用户安装后重启。
下载按钮在缺失关键依赖时仍可点击,但提示已提前给出,避免无谓失败。
